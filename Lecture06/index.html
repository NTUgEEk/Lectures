Multithreading

並行處理

  • Pros: 不會被卡死
  • Cons: 大家一起慢 + 不停切換

Concurrency

一次處理一個

  • Pros: 切換次數少
  • Cons: 耗時計算會卡死

SQL v.s. NoSQL

Wiki 上也有詳盡的解釋。一言以蔽之，SQL 是傳統的 Relational Database ，並使用 SQL 語句來操作。而 NoSQL 是指用其他方式來儲存資料的 Database ，而且通常都不是用 SQL 語句來操作。
